JavaScript: Novice to Ninja

Future Functions

Function Properties and Methods

  • functions are first-class objects so they can have properties and methods themselves
  • the call method can be used to set the value of this inside a functions to an object that is provided as the first argument
  • the call and apply methods allow generalized functions to be written that are not tied to specific objects by being objects of that object
  • memoization (caching)

Immediately Invoked Function Expressions

  • invoked as soon as it is defined
  • the function has to be made into an expression (place the whole declaration inside parantheses and then place parentheses after it
  • this can be used to enclose a block of code inside its own private scope so it doesn't interfere with any other part of the program

Functions that Define and Rewrite Themselves

  • a function can also define itself and redefine itself
  • to do so, assign an anonymous function to a variable with the same name as the function
  • this is called the Lazy Definition Pattern
  • can be used when some initilization is required the first time it's called

Callbacks

  • the setTimeout method accept a callback to a function as its first parameter and a number of milliseconds as its second parameter
  • promises simplify the callback promise
  • when a promise is settled, you can use the then method to deal with the outcome
  • the catch method can be used to specify what happens if it fails
  • async functions allow you to write asynchronous code as if it was synchronous
  • functions can return a function
  • to use a returned function, it needs to be assigned to a variable

Closures

  • a closure is a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program
  • a closure is formed when the inner function is returned by the outer function

Functional Programming

  • pure functions help make functional programming more concise and predictable than in other programming styles
  • functions should perform a single task
  • high-order functions are functions that accept another functions as an argument, or return another function as a result, or both
  • currying is a process that involves the partial application of functions
  • currying allows you to turn a single function into a series of functions instead

Ajax

The Fetch API

  • the Fetch API superseded XMLHttpRequest
  • fetch returns a promise
  • JSON is the most common format for AJAX responses
  • others are text and blob
  • you can use Headers, Request and Response Objects

FormData

  • the Fetch API includes the FormData interface, which makes it much easier to submit informations in forms using Ajax
  • this interface really helps when a form contains files to upload

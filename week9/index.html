The Window Object

  • every javascript environment has a global object

The Browser Object Model

  • the Browser Object Model is a collection of properties and methods that contain information about the browser and computer screens
  • JavaScript can be run in different environments; the BOM only makes sense in a browser environment

The Document Object

  • cookies are small files that are saved locally on a user's computer
  • the document.cookie property acts like a speical type of string
  • assigning another cookie to it won't overwrite the entire property, it will append it to the end of the string

Timing Functions

  • The window object provides some useful methods for scheduling the exectuion of a function, and for repeatedly executing functions at regular intervals.
  • The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter.
  • The window.setInterval() method works in a similar way, except it repeatedly invokes the callback function every given number of milliseconds.
  • Be careful using this with either method, as the binding of this is set to the window object, rather than the method's object.
  • setTimeout() and setInterval() methods can be used to animate elements of a webpage
  • requestAnimationFrame makes it smoother
  • Generally it is better to use CSS for any animation effects.

HTML5 APIs

The data- Attribute

  • The data- attribute is a way of embedding data in a web page using custom attributes that are ignoted by the browser.
  • They're also private to the page. They're sole purpose is to be used by a JavaScript program.
  • If a data- attribute's name contains hyphens, they are replaced with camel-case notation (data-max-lenghth -> dataset.maxLength)
